Crimp Installation

This is a Basic guide to installing CRIMP and assumes that your webserver is configured according to the default layout as shown within the CRIMP requirements documentation.

The CRIMP install should be unzipped or untared into your web server root (your home directory), if your webserver or webhost is set up differently then please read the advanced installation documentation.

CRIMP Packaging

CRIMP is packaged in a compressed file and includes basic modules with an example configuration to get you started. These packages are usually provided in 2 formats, either a ZIP or a TARBALL. Download the appropriate one for your requirements depending on your type of install.

Installing over FTP

For this type of install, your downloaded module in either ZIP or TARBALL format needs to be extracted into a temporary directory on your local machine before uploading to your web server.

From Windows

Using your favorite compression utility (example: Winzip), extract all the files to a temporary directory and open your FTP client and connect to your server host (or use windows file manager by typing 'ftp://[username]@[server-host.com]' into the address bar, you will be prompted for your password.).
Select your extracted CRIMP directory and upload it in its entirety into your remote webserver home directory on your server host.

*please note, connection problems are not discussed in this document. Please check your firewall settings!

Once CRIMP is uploaded, you will need to check the file permissions on the uploaded files according to the section later within this page.

From Linux

Untar the tarball into a temprary directory:

    mkdir ~/temp
    cd ~/temp
    tar -xjpvf [ModuleName].tar.bz2    # not sure if this is going to create ~/temp/Crimp


Use your FTP client  to upload the extracted CRIMP files into your remote webserver home directory.

Once CRIMP is uploaded, you will need to check the file permissions on the uploaded files according to the section later within this page.

Installing over SSH

If you have SSH access to your webserver, then installing CRIMP over SSH is the prefered method. Windows users can use a SSH client (Example: Putty which includes pscp.exe) and linux users can use their console.

Upload CRIMP and MD5 sum

Windows

If you have downloaded CRIMP and its MD5 sum onto your local windows machine, you can use pscp.exe in your command console to upload the files.
(Start - Run : then type 'cmd' for Windows XP and NT or type 'command' for Windows 98/98se/ME)
Change to your Putty install directory by typing:

    cd [path\to\putty]
    pscp [path\to\Crimp-version.tar.bz2] [username]@[server-host.com]:[webserver root]
    pscp [path\to\Crimp-version.tar.bz2.sum] [username]@[server-host.com]:[webserver root]

Then open your SSH client and connect to your server host.

Linux

If you have downloaded CRIMP and its MD5 sum onto your local linux machine, you can use scp in your console to upload the files.

    scp [server-host]:~/ ~/[Crimp-version.tar.bz2]
    scp [server-host]:~/ ~/[Crimp-version.tar.bz2.sum]
    ssh [username]@[server-host]

SSH Client

Once logged into your server host over SSH, you can run the following commands to install CRIMP:

    cd ~/
    # MD5 check the module for consistancy
    md5sum -c [Crimp-version].tar.bz2.sum
    # If the check returns OK then unpack the tarball
    tar -xjpvf [Crimp-version].tar.bz2

CRIMP is now installed and ready for configuration.

File Permissions

Depending on how CRIMP was installed, it is recomended that you check the file and directory permissions and set them according to your webservers requirements.

public_html (directory)

	crimp_assets			world readable

cgi-bin

	index.pl				world readable and executable

		Crimp (directory)
			var (directory)	world readable and writable

Configuration

Once CRIMP is installed into the webserver root, rename the crimp.ini.example file as crimp.ini which is located in the cgi-bin directory of your install.

You can now test the installation by pointing your browser to your website URL.
