WordLink

This is a concept idea for a module yet to be developed by the crimp team.

WordLink parses the chosen page content and makes links of certain key words (similar to wiki words).

Firstly, the module would have to appear directly above whatever module is being used to grab the page within the config file.

example crimp.ini

[/docs]
WordLink = [path to writable text file]
ContentDirectory = ../cgi-bin/Crimp/docs

Secondly, for users that have minimum knowlege of their website, a text file should be included to take care of the initial setup.

example wordlink.txt (probably in 'var' and set to chmod 755)

CRIMP = http://crimp.sourceforge.net
WordLink = http://crimp.sourceforge.net/docs/modules/WordLink
IND-Web = http://www.ind-web.com/
XYZ = http://www.xyz-network.com/

Now the crimp.ini file and initial writable text file has been set, any pages that contain the word CRIMP (within the chosen section of the crimp.ini) will be have that word replaced with <a href='http://crimp.sourceforge.net'>CRIMP</a>

There are several ways to add words to the list:

  1. Edit the text file manually
  2. Let the module do it for you

To let the module add words to the list, all the user would have to do is enter the word into a page as follows: 

[[CRIMP|http://crimp.sourceforge.net]]
[[my computer|http://my.site.net/my-computer.html]]
Any pages containing these terms will then have the link automatically inserted.

Possible problems and solutions

The initial users page with the WordLink may not be writable and so the WordLink will always be in the file as written by the user. WordLink should be able to overcome this by rewriting the text on the fly and not entering the word twice into the WordLink text file.

There will be a page slowdown as WordLink has to check every word in its file against the page that has been grabbed. I dont think that this will be too noticable on todays servers and if a user doesnt like the slowdown, then they shouldnt use the module.
