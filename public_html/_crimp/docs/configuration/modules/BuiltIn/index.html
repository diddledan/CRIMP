CRIMP Built In Modules

CRIMP has been designed to work without the use of module plugins to give some basic functionality.
A few settings can be tweaked and activated using built in configuration commands.

Command List

The commands take the form of ModuleName = Value in the crimp.ini file.

CgiDirectory Can only be used within the root section of the crimp.ini file
../cgi-bin (default)
This is the location of the cgi-bin directory and is not normally needed to be changed.
Debug
 Can only be used within the root section of the crimp.ini file
on (default)
Prints the output of CRIMP Debug at the end of a page.
off
Switches CRIMP Debug mode off entirely.
page
Only displays CRIMP Debug mode if called from the browser by appending ?debug=on to the url.
DescriptionMeta Some indexing engines look for META elements that give a short description. The Meta Description Tag allows you to influence the description of your page for crawlers that support the tag and can be set within the crimp.ini file on a global or per directory basis.
Your descriptive sentence or two goes here. (example)
Generally, 200 to 250 characters may be indexed, though only a smaller portion of this amount may be displayed.
ErrorDirectory Can only be used within the root section of the crimp.ini file
../cgi-bin/Crimp/errors (default)
This is the location of the errors directory and contains html pages of error codes and their error messages. If you prefer to use custom error pages, this can be changed to the new path containing the custom pages. It is not recomended you alter the files within the default directory as they could be overwritten while updating crimp.
FirstRun Can only be used within the root section of the crimp.ini file
on (default)
After following the CRIMP install guide, this command appears in the new crimp.ini file.
Unless you need CRIMP to do its post install checks then it is safe to delete this config entry.
HtmlDirectory Can only be used within the root section of the crimp.ini file
../public_html (default)
This is the location of the public_html directory and is not normally needed to be changed.
KeywordsMeta Some indexing engines look for META elements that define a comma-separated list of keywords/phrases. The Meta Keywords Tag is sometimes useful as a way to reinforce the terms you think a page is important for ON THE FEW CRAWLERS THAT SUPPORT IT and can be set within the crimp.ini file on a global or per directory basis.
your keywords,go here,separated by a comma,but not a space (example)
Generally, 25 words or phrases, with each word or phrase separated by commas.
PageTitle Not entirely sure if this is needed.
PluginOrder Controls the plugin execution order of modules. CRIMP modules are called alphabetically by default but usually content creation modules are required to be called before any other module.
VirtualRedirect,BreadCrumbs,RandomContent
This would make sure the VirtualRedirect module is called before the BreadCrumbs and RandomContent modules instead of alphabetical order.
(Please note that DocumentTemplate and ButtonBar are always called last if within the global section or any per directory section of the crimp.ini file and are not needed within the comma separated list.)
RobotsMeta Search engine control of 'robots implementing The Robots META tag' which allows HTML authors to indicate to visiting robots whether a document may be indexed, or used to harvest more links. When used on a per directory basis, enables control of visiting spiders and bots.
The list of terms in the content is ALL, INDEX, NOFOLLOW, NOINDEX. The name and the content attribute values are case insensitive.
index,follow (default)
A robot is able index this document, and analyse it for links.
noindex,nofollow
A robot should neither index this document, nor analyse it for links.
VarDirectory Not sure if this is configurable... It shouldnt need to be changed.
