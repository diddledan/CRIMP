Module Installation

This is a Basic guide to installing 3rd party modules for CRIMP.

CRIMP Modules not included with the default CRIMP install should be unzipped or untared into your cgi-bin directory. Depending on your web server or web host, these files and directories should be world readable unless stated within the modules own documentation.

Module Packaging

Modules can be downloaded from 3rd party websites and are usually packaged in a compressed file. These packages are usually provided in 2 formats, either a ZIP or a TARBALL. Download the appropriate one for your requirements depending on your type of install.

Installing over FTP

For this type of install, your downloaded module in either ZIP or TARBALL format needs to be extracted into a temporary directory on your local machine before uploading to your web server.

From Windows

Using your favorite compression utility (example: Winzip), extract all the files to a temporary directory and open your FTP client and connect to your server host (or use windows file manager by typing 'ftp://[username]@[server-host.com]' into the address bar, you will be prompted for your password.).
Select your extracted CRIMP directory and upload it in its entirety into the remote cgi-bin folder on your server host.

*please note, connection problems are not discussed in this document. Please check your firewall settings!

Once the module is uploaded, you can view its documentation from either your website ( http://[yoursite.com]/docs/modules/[ModuleName]/install.html ) or on your local windows machine ( [temporary directory]\Crimp\docs\[ModuleName]\install.html ).

After checking the file permissions on the uploaded files listed in the module install.html, you are now ready to configure it for use within your website.

From Linux

Untar the tarball into a temporary directory:

    mkdir ~/temp
    cd ~/temp
    tar -xjpvf [ModuleName].tar.bz2    # not sure if this is going to create ~/temp/Crimp

Use your FTP client  to upload the CRIMP directory into your cgi-bin.

Once the module is uploaded, you can view its documentation from either your website ( http://[yoursite.com]/docs/modules/[ModuleName]/install.html ) or on your local linux machine ( ~/temp/Crimp/docs/[ModuleName]/install.html ).

After checking the file permissions on the uploaded files listed in the module install.html, you are now ready to configure it for use within your website.

Installing over SSH

If you have SSH access to your webserver, then installing modules over SSH is the prefered method to add 3rd party modules to CRIMP. Windows users can use a SSH client (Example: Putty which includes pscp.exe) and linux users can use their console.

Upload the Module and MD5 sum

Windows

If you have downloaded the Module and MD5 sum onto your local windows machine, you can use pscp.exe in your command console to upload the files.
(Start - Run : then type 'cmd' for Windows XP and NT or type 'command' for Windows 98/98se/ME)
Change to your Putty install directory by typing:

    cd [path\to\putty]
    pscp [path\to\ModuleName.tar.bz2] [username]@[server-host.com]:[path\to\cgi-bin]
    pscp [path\to\ModuleName.tar.bz2.sum] [username]@[server-host.com]:[path\to\cgi-bin]

Then open your SSH client and connect to your server host.

Linux

If you have downloaded the Module and MD5 sum onto your local linux machine, you can use scp in your console to upload the files.

    scp [server-host]:~/[path/to/cgi-bin] ~/[path/to/ModuleName.tar.bz2]
    scp [server-host]:~/[path/to/cgi-bin] ~/[path/to/ModuleName.tar.bz2.sum]
    ssh [username]@[server-host]

SSH Client

Once logged into your server host over SSH, you can run the following commands to install your module:

    cd ~/[path/to/cgi-bin]
    # If you have not yet downloaded the module
    wget http://[3rd-party-module-host]/[path/to/ModuleName.tar.bz2]
    wget http://[3rd-party-module-host]/[path/to/ModuleName.tar.bz2.sum]
    # Installation
    # MD5 check the module for consistancy
    md5sum -c [ModuleName].tar.bz2.sum
    # If the check returns OK then unpack the tarball
    tar -xjpvf [ModuleName].tar.bz2

The module is now installed and ready for configuration.

Configuration

Once the module is installed into ../cgi-bin/Crimp, the corisponding docs will be installed into the appropriate place in ../cgi-bin/Crimp/docs which is accessable by your website if configured in ../cgi-bin/crimp.ini

For further configuration of the module, check the module documentation at http://[yoursite.com]/docs/modules/[ModuleName]/configuration.html.
