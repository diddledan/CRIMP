CRIMP Configuration

Configuring CRIMP is all done within its easy to use crimp.ini file.

There are 2 main sections to think about when configuring a CRIMP website,

  1. What modules you want to use globally.
  2. What modules you would like to use on a per web directory basis.

By using the examples within ../cgi-bin/crimp.ini, it is possible to create page layouts to suit your needs.

The Global Section

This section is at the top of the crimp.ini file and allows you to add modules to your entire site. 

# Global Options
# all files must default to standard redirection
# default options configure files for the local server on
 
# Print debug output at the bottom of the page [on/off/page]
DebugMode = on
 
# Plugin Execution Order
# LEAVE OUT DocumentTemplate - this will automatically be called last
PluginOrder = UserAuth,ContentDirectory,ContentDatabase,BreadCrumbs,FileList,SiteMenu
 
# The template used for the entire site
DocumentTemplate = ../template/default-crimp.html

Lines starting with a # are comments and are there to assist you in the configuration of CRIMP.

The Directory Sections

These sections allow you to set specific parts of your website to use different modules to create content and will overide any global options if needed.

# Website root (http://your-site/)
[/]
PluginOrder = ContentDirectory,FileList,BreadCrumbs
FileList = horizontal
BreadCrumbs = top
ContentDirectory = ../content/

# Website docs (http://your-site/docs/)
[/docs]
PluginOrder = ContentDirectory,FileList,BreadCrumbs
FileList = horizontal
BreadCrumbs = top
ContentDirectory = ../cgi-bin/Crimp/docs/

Calling Modules

Modules are called within the crimp.ini file from the relevent sections by using the following format:

ModuleName = Module Value

The most important module within the crimp.ini file is called Debug, this is a built in module and will assist you in configuring your CRIMP website. The Debug module is explicitly set within the global section of the crimp.ini file and has 3 settings: 

# permanently activate debug throughout the website
Debug = on
 
# view debug by appending ?debug=on to your URL
# eg http://my-site?debug=on
Debug = page

# switch debug off throughout your website
Debug = off

A complete list of all modules that are not built in and are available to CRIMP can be seen within the output of CRIMP Debug. You can find instructions on howto configure these plugin modules for use in your site by selecting the module name within the CRIMP modules section.
