500 - Server Error

A FATAL ERROR has occured, CRIMP cannot continue.

CRIMP could not read the config.xml file in public_html/_crimp/config.xml .

Possible Reasons and Solutions

  1. This is a new install of CRIMP and the crimp.ini file is missing.
    • Copy the crimp.xml.example file to crimp.xml within the public_html/_crimp directory.
  2. The permissions on the config.xml file are incorrect.
    • If utilising Apache's mod_suphp or similar facility that allows the server to execute CRIMP as your local user id, then the file must at least be readable by your userid.
    • For all other scenarios, the config.xml file must be world readable (on *nix: chmod 644 config.xml)
